## Task 1 — Inheritance in Java (25 minutes)

Watch the following videos on Inheritance in Java:

- 📹 [Inheritance in Java — https://www.youtube.com/watch?v=dFuVh_Bzy9c]
  

- 📹 [The super keyword in Java — https://www.youtube.com/watch?v=Qb_NUn0TSAU&t=289s]
  
While watching, refer to **lesson.md Part 2** and pay attention to:
- How the `extends` keyword creates a parent-child relationship between classes
- How `super()` is used to call the parent class constructor
- What the `protected` access modifier allows compared to `private`
- Why Java does not support multiple inheritance

**Guiding Questions:**
1. What is the difference between a parent class and a child class?
2. What does `super(name, birthYear)` do inside a child class constructor?
3. Why would you use `protected` instead of `private` for a field?

## Task 2 — Polymorphism in Java (20 minutes)

Watch the following video on Polymorphism in Java:

- 📹 [Method Overloading in Java — https://www.youtube.com/watch?v=KpwBVAYbPDA]
  

- 📹 [Method Overriding in Java — https://www.youtube.com/watch?v=CLzgS08equQ]
 

While watching, refer to **lesson.md Part 3** and pay attention to:
- The difference between overloading (same name, different parameters) and overriding (redefining a parent method)
- Why overloading is decided at compile time and overriding at runtime
- What the `@Override` annotation does and why it is good practice to use it
- How `super.methodName()` lets you call the parent version of an overridden method

**Guiding Questions:**
1. What is the difference between method overloading and method overriding?
2. What happens if you misspell an overriding method name and forget to use `@Override`?
3. When would you use `super.doWork()` inside an overriding method?

## Task 3 — Abstract Classes and Interfaces (25 minutes)

Watch the following videos on Abstraction in Java:

- 📹 [Abstract Classes in Java —https://www.youtube.com/watch?v=HvPlEJ3LHgE&t=256s]
  


  

While watching, refer to **lesson.md Part 4** and pay attention to:
- What makes a class abstract and why it cannot be instantiated
- The difference between an abstract method and a regular method
- How an interface differs from an abstract class
- Why a class can implement multiple interfaces but only extend one class

**Guiding Questions:**
1. What is the difference between an abstract class and an interface?
2. Can an abstract class have non-abstract methods? Can an interface?
3. When would you choose to use an interface over an abstract class?
